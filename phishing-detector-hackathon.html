Over 3.4 billion phishing emails are sent every day. Existing detectors are black boxes — they block emails with no explanation, leaving users confused and IT teams blind.

🧠

Your Solution

PhishScan AI detects and explains — giving users, developers, and security teams actionable insight into every flagged email.

03 — System Architecture

How It Works Under the Hood

A two-layer pipeline: fast rule-based pre-screening followed by a deep NLP model, with an XAI layer generating explanations.

📧 Email Input Raw .eml / text
→
🔍 Preprocessor Header · Links · Body
→
📏 Rule Engine Domain · Keywords · URLs
→
🤖 NLP Model TF-IDF / BERT
→
💡 XAI Layer LIME / SHAP
→
📊 Report Score + Reasons

Layer 1 — Rule-Based Engine

Fast, interpretable checks that catch obvious red flags before hitting the model.

  • Domain similarity scoring (Levenshtein distance)
  • Urgency / fear keyword lexicon
  • URL redirect chain analysis
  • Header authentication checks (SPF, DKIM)
  • Sender ↔ brand domain mismatch

Layer 2 — NLP Classification

Trained on labeled email datasets. Understands context, not just keywords.

  • TF-IDF + Logistic Regression (fast baseline)
  • Fine-tuned BERT for semantic understanding
  • LIME/SHAP to highlight which tokens drove prediction
  • Ensemble score combining rule + ML outputs
  • Confidence threshold calibration

05 — 10-Day Build Plan

From Idea to Demo in 10 Days

A realistic, phased plan that ensures you always have a working demo — even if later features aren't finished.

1-2DAYS

Data Collection & Setup

  • Download Enron Spam Dataset (~33k emails) and SpamAssassin corpus
  • Set up Python environment, install dependencies
  • Parse raw .eml files into structured DataFrame (subject, body, sender, headers)
  • Exploratory data analysis — class balance, common keywords, domain patterns
3-4DAYS

Rule Engine + Baseline ML Model

  • Build rule-based detector: domain spoof detection, urgency lexicon, URL analysis
  • Train TF-IDF + Logistic Regression as baseline (target: ~94% accuracy)
  • Write unit tests for each rule; log which rules trigger most
  • Build feature importance visualization for baseline model
5-6DAYS

BERT Fine-Tuning + Explanation Layer

  • Fine-tune DistilBERT on the email dataset (lighter & faster than full BERT)
  • Integrate LIME to generate token-level explanations per prediction
  • Build ensemble scorer combining rule engine + ML output into a 0–100 risk score
  • Calibrate confidence thresholds; handle edge cases (short emails, non-English)
7-8DAYS

Streamlit Web Interface

  • Build the main UI: paste email / upload .eml file → get instant analysis
  • Animated risk score gauge, color-coded verdict badge
  • Interactive breakdown: which words/phrases triggered each reason
  • History tab: see past scanned emails with scores
9-10DAYS

Polish, Testing & Presentation

  • Test on 20+ real phishing email samples from PhishTank
  • Record a 2-min demo video as backup for live demo failures
  • Prepare pitch deck: problem → solution → demo → impact → future scope
  • Deploy to Streamlit Cloud (free) for a live shareable URL
